Setting up your workspace

Before starting this tutorial make sure that you have Sublime text downloaded (or any other text editor if you have one) and Google Chrome.These tools will help you build your page and debug any errors you may have along the way.

First thing we need to do is create a folder for our webpage. On your desktop or in your documents folder create a new folder and name it "website", or any other name you like. Inside this folder you're going to create two more folders and name them "css" and "img". These two folders will help you keep your page organized by storing images in the img folder and your stylesheets in the css folder.

Now open up Sublime and select File > Open [or use CMND + O if you are on a Mac - CTRL +O if you are using Windows]. Select your "website" and click open to open the whole project folder in sublime. When you have done this your sublime text editor should look like this:

All that's left to do is create our html and css files. Select File > New [CMND+N on Mac, CTRL+N on Windows] to create a new file. For now just save this file in your "website" folder and name it "page.html". To create your css file, create a new file and save it in your css folder with the name "styles.css".

Creating Your HTML Document

Html organizes code by using tags that look something like this <html>. All content in an html document must be surrounded by an opening tag and it's corresponding closing tag, i.e <html></html>. To get our document organized so that a browser can understand it's content, create the following structure in your html page.
A note: Html ignores whitespace, but proper indenting makes code much more readable and can help you find errors quicker, just press the 'tab' key to create the space indenting

<!DOCTYPE html>
<html>
<head>
</head>
<body>
</body>
</html>
Linking your style sheet

Even though there are no styles yet, it is important to link your style sheet to your html document at the start. This allows you to write your css in another file, keeping your code organized, while still being able to style the look of your page. So, inside your <head> tags add the following line of code.

...
<head>
<link rel="stylesheet" href="css/styles.css" >
</head>
...

The href attribute in your link tag references the location of your style sheet based on the location of your html file. You'll also notice that the link tag does not have a closing </link>. Though this breaks the normal convention of html it's considered a shorthand way of writing this tag. The reason being that you never actually write anything or place any additional tags between <link> </link>. Because of this a browser understands not to require a closing tag. As a general rule, if you see this type of behaviour, it is normally tags that never have content written inside of them that are treated this way. Another example is the <img> tag.

That's it! Now you can go and start building your page.

Building the top

Before we get started, we're going add a couple styles to format the document. These styles clear up some default margins and apply some standard fonts and colours. You can always google some the names to find out more specifics. It's important to note the structure of CSS. The element you are targetting for styling is written out in full and the properties you give it are contained within the { } brackets. Properties are written out in the format propertyname:property;. Note: semicolons denote the end of line like a period does for a sentance. A full list of properties can be found here
So, in your styles.css document, add the following lines at the top:

html,body{
height:100%;
margin:0;
background: #d8d8d8;
}
body{
color:#58595b;
font-family: "europa";
font-weight: normal;
font-style: normal;
font-size: 100%;
line-height: 1.5;
position: relative;
cursor: auto;
letter-spacing: 0.05rem;
-webkit-font-smoothing: subpixel-antialiased
}
Background Image

First thing you are going to do is create a large background image that sits in its position while you scroll. Find an image you like and save it in your "img" folder. In your Html you're going to add the following tag between the Body tags:

...
<body>
<section class="top"></section>
</body>
...

This will be our first introduction to using Id targeted styles in CSS. We gave the section tag an id called top. In our CSS file we're going add the following lines of code. The hashtag symbol denotes that we are going to be adding styles for elements with the id of "top".

#top{
background:url('../img/empire.jpg');
background-attachment: fixed;
background-position: bottom;
background-size:cover;
height:60%;
width:100%;
}

CSS syntax is somewhat easy to understand since the property names generally represent what they do. So, looking at background:url, you are going to replace empire.jpg with the name of your image. The '../' helps direct the path to the image. The image is in the img folder and the style sheet is in the css folder, using '../' tells the code to go back one folder to enter the website folder (referred to by programmers as the root folder) and then forwards into the img folder. Now if you double click your page.html file, it should open in the browser displaying your image.

We're going to add another tag which will represent the contents of our resume. This will be referred to as a container since it will contain the next few blocks of code.Place the following line under the previous section.

With clearfix and profile, you can see the first examples of class targetting with styles. It is typically more beneficial to use classes to style our elements than Id's. The reason being that elements can only have one Id but they can have multiple classes. This way we can add many different styles to one element through different classes. Classes are denoted with a period as a prefix in CSS. Note: Clearfix is a style that helps parent elements clear their child elements, you can read more about it here

.clearfix:after {
visibility: hidden;
display: block;
font-size: 0;
content: " ";
clear: both;
height: 0;
}

In this block we can see the first set of tags that have actual text between them. There is also an img tag which is another example of a tag that doesn't require a closing tag. In the src attribute, replace the image with a picture of yourself that you will save into the img folder. Once you're done, we are going to add these styles to the classes above:

There are a couple examples of more advanced CSS in the code above. The use of absolutely positioned elements allows us to take advantage of centering the container with left and right percentages. Since we know the width of the container is 25em, we can move it to the left half it's width to center it perfectly. You can read more on centering elements here

Adding your personal information

In this step we are going to create our first reusable block of code. Using html and css we will be able to apply this block of code to style multiple elements of text. Just below the avatar-container add this code:

<span class="title">Contact Information</span>
<p>
647.518.5633 <br>
r@gmail.com <br>
http://rares.com <br>
</p>
</div>
</article>

Add the following styles as well:

.resume-left {
float:left;
width:50%;
margin-top:200px;
}
#module{
margin-bottom:30px;
}
#module .title,#module .sub-title{
color:#4fbfa6;
font-size:14pt;
font-weight:400;
}
#module .sub-title{
font-size:10pt;
}
#module p{
font-weight:100;
font-size:10pt;
line-height:12pt;
letter-spacing: 0.1rem;
word-spacing: 0.1rem;
text-rendering: optimizeLegibility;
-webkit-font-smoothing: subpixel-antialiased
}

Above you can see how to write css to apply the same set of properties to different classes. Farther down on the second and third set of styles you can see css's cascading style nature (CSS stands for Cascading StyleSheet). The font-size in the second assignment of .sub-title overwrites the first font-size of 14pt. The .sub-title class still inherits the color and the font-weight, but this way it is also able to have a different font-size. The last feature of CSS you can see in the code above is an example of CSS nesting. Nesting ensures that the properties are only applied to the class when they are a child of a parent element with the leading class or Id (it is an Id in this case). You can read more about nesting here
